Vision Document

Project Name · Domain · Problem Statement · Vision · Scope

📌 Project Name

DormHub – Smart Dormitory Management and Issue Tracking System

🏛️ Domain

Student housing management – maintenance requests, payments, user management, and AI-powered document processing.

⚠️ Problem Statement

University dormitories manage student issues through paper forms, in-person visits, phone calls, or scattered messaging apps. This leads to lost requests, no transparency, delayed responses.

👁️ Vision

A web-based intelligent platform that digitizes and automates dormitory issue management using AI .

✅ Scope (Included)

  • Student registration and role-based login
  • Submitting maintenance requests with images
  • Tracking request status in real-time
  • Support team reviewing and prioritizing requests
  • Technician viewing assigned tasks
  • Student final confirmation
  • Manager viewing reports
  • AI processing of documents
  • Online payment
  • 2FA and activity logging

🚫 Scope (Not Included)

  • Real-time chat
  • IoT/smart device integration
👥

User Roles (6 Roles)

RBAC implementation with role-based permissions

🎓 Student

Submit issues, upload images, track requests, confirm resolution, pay fees.

🔧 Technician

View assigned requests, update status, log work details.

🎧 Support

Review requests, prioritize, assign to technicians.

📊 Manager

View all requests, generate reports, approve costs, upload documents for AI .

👑 Admin

Manage users, roles, settings, view logs.

👀 Guest

View public dormitory information (no login).

Code Level (Level 4) — Core Domain Models

Essential classes, attributes, methods, and relationships

<?php

// ==================== ENUMS ====================

enum Role
{
    case STUDENT;
    case TECHNICIAN;
    case SUPPORT;
    case MANAGER;
    case ADMIN;
}

enum RequestStatus
{
    case PENDING;      // waiting for support review
    case IN_PROGRESS;  // assigned to technician
    case COMPLETED;    // technician finished
    case CONFIRMED;    // student confirmed
}

enum Priority
{
    case LOW;
    case MEDIUM;
    case HIGH;
}

// ==================== CLASS: User ====================

class User
{
    private string $id;
    private string $fullName;
    private string $email;
    private string $passwordHash;
    private Role $role;
    private ?string $roomNumber;
    private bool $isActive;

    public function register(string $email, string $password, string $fullName): User {}
    public function login(string $email, string $password): string {}
    public function sendOTP(string $email): void {}
    public function verifyOTP(string $code): bool {}
    public function hasPermission(string $action): bool {}
}

// ==================== CLASS: MaintenanceRequest ====================

class MaintenanceRequest
{
    private string $id;
    private string $trackingNumber;
    private string $title;
    private string $description;
    private string $category;
    private Priority $priority;
    private RequestStatus $status;
    private string $studentId;
    private ?string $technicianId;
    private array $imageUrls;
    private ?string $workNotes;

    public function submit(string $studentId, array $data): MaintenanceRequest {}
    public function assign(string $technicianId): void {}
    public function startProgress(): void {}
    public function complete(string $notes): void {}
    public function confirmByStudent(): void {}
    public function addImage(string $url): void {}
}

// ==================== CLASS: Payment ====================

class Payment
{
    private string $id;
    private string $userId;
    private float $amount;
    private string $title;
    private string $status;  // PENDING, PAID, FAILED
    private ?string $transactionId;
    private ?string $receiptUrl;

    public function initiate(float $amount, string $title, string $userId): Payment {}
    public function verify(string $transactionId): bool {}
    public function generateReceipt(): string {}
}

// ==================== CLASS: ActivityLog ====================

class ActivityLog
{
    private string $id;
    private string $userId;
    private string $action;
    private string $ipAddress;
    private \DateTime $createdAt;

    public function log(string $userId, string $action, array $details, string $ipAddress): void {}
    public function getLogs(array $filters): array {}
}

// ==================== RELATIONSHIPS ====================
// User (1) ---- (*) MaintenanceRequest
// MaintenanceRequest (1) ---- (1) Payment
// User (1) ---- (*) ActivityLog
